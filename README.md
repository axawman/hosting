Автоматизированная PaaS-платформа (Platform as a Service), предназначенная для мгновенного развертывания и хостинга пользовательских веб-проектов в изолированной инфраструктуре. Система обеспечивает автоматическое управление контейнеризацией, распределение сетевых ресурсов и мониторинг состояния сервисов.

## Основные функциональные возможности

- **Автоматизированный деплой**: Загрузка ZIP-архивов с веб-ресурсами (HTML, CSS, JS), их автоматическая распаковка и запуск в изолированных контейнерах.
- **Динамическое управление ресурсами**: Использование Docker SDK для программного создания, остановки и удаления пользовательских контейнеров в реальном времени.
- **Интеллектуальный учет портов**: Автоматическое выделение свободных сетевых портов из заданного пула и строгий контроль их жизненного цикла.
- **Разграничение прав доступа**:
    - **Личный кабинет студента**: Инструменты управления собственными проектами, мониторинг статуса и просмотр назначенных портов.
    - **Панель управления преподавателя**: Глобальный мониторинг всех ресурсов, управление реестром портов и возможность принудительной остановки сервисов.
- **Корпоративная безопасность**: Регистрация и доступ к платформе ограничены исключительно почтовыми доменами университета (@volsu.ru, @master.volsu.ru).
- **Темная тема в стиле GitHub**: Современный минималистичный интерфейс, оптимизированный для длительной работы.

## Технологический стек

- **Ядро системы**: Python 3.12, FastAPI.
- **Инфраструктура**: Docker и Docker Compose для полной контейнеризации платформы.
- **Оркестрация**: Docker SDK для Python (управление пользовательскими сайтами).
- **База данных**: PostgreSQL 15 (хранение метаданных проектов и реестра портов).
- **Веб-сервер**: Nginx (основной шлюз и runtime-образ nginx:alpine для сайтов).
- **ASGI-сервер**: Uvicorn (запуск приложения FastAPI).

## Архитектура системы

Текущая архитектура проекта реализована в виде единого приложения в директории /app, включающего следующие компоненты:
- **main.py**: Основной файл приложения FastAPI, содержащий маршруты и обработчики запросов.
- **docker_manager.py**: Модуль для управления Docker контейнерами студентов.
- **db.py**: Конфигурация базы данных и ORM (SQLAlchemy).
- **templates**: Директория с HTML-шаблонами для веб-интерфейса.

## Установка и развертывание

Проект полностью адаптирован для быстрого развертывания через Docker.

1. **Клонирование репозитория**:
   
   ```bash
   git clone https://github.com/axawman/hosting
   cd hosting
   ```
3. **Настройка переменных окружения .env на основе .env.example (изменив параметры БД, секретный ключ и диапазон портов)**
   ```bash
   cp .env.example .env
   ```
   отредактировать данные
4. **Запуск платформы**:
   ```bash
   docker compose up -d --build
   ```
   После запуска сервис доступен по адресу http://localhost
## Безопасность и надежность

- **Изоляция среды**: Каждый пользовательский сайт работает в отдельном контейнере в режиме read-only.

- **Автоматизированный CI/CD**: Пайплайн через GitHub Actions для тестирования и автоматического деплоя на VPS по SSH.

- **Система резервного копирования**: Экспорт базы данных и медиафайлов в ZIP-архив для миграции или восстановления.


## Объяснение принципа работы

Пользователь открывает главную страницу. Если он не авторизован, сайт перекрывает окно входа/регистрации. После входа обычный пользователь видит только свои проекты: может загрузить ZIP, платформа распакует его, создаст Docker-контейнер с `nginx:alpine`, положит туда файлы сайта и покажет проект в списке.

Администратор вместо обычной страницы видит админ-панель: пользователей, их лимиты, контейнеры, Docker-статус и кнопки управления контейнерами.

**app/main.py**
Главный файл приложения.

Он отвечает за:
- создание FastAPI-приложения;
- маршруты страниц и форм;
- регистрацию, вход и выход;
- проверку текущего пользователя по cookie-сессии;
- отображение обычной панели или админ-панели;
- загрузку ZIP-проекта;
- проверку лимитов пользователя;
- удаление проекта;
- админские действия: смена лимитов, роли пользователя, запуск/остановка/перезапуск контейнеров.

Ключевой поток такой:
- `GET /` показывает страницу;
- `POST /register` создает пользователя;
- `POST /login` авторизует;
- `POST /logout` выходит из аккаунта;
- `POST /deploy` загружает и запускает проект;
- `POST /delete/{project_id}` удаляет проект;
- `/admin/...` маршруты доступны только админу.

**app/db.py**
Файл базы данных и моделей.

Он отвечает за хранение:
- пользователей `User`;
- сессий входа `UserSession`;
- проектов `Project`;
- глобальных настроек админа `AdminSettings`.

Также здесь:
- подключение к PostgreSQL через `DATABASE_URL`;
- fallback на SQLite, если переменная не задана;
- создание таблиц при старте;
- хеширование и проверка паролей;
- создание админа по умолчанию из `ADMIN_EMAIL` и `ADMIN_PASSWORD`.

Пароли не хранятся открытым текстом: они хешируются через `pbkdf2_hmac`.

**app/docker_manager.py**
Файл для работы с Docker.

Он отвечает за:
- проверку доступности Docker;
- получение списка student-контейнеров;
- создание контейнера проекта;
- загрузку файлов сайта внутрь контейнера;
- удаление контейнера;
- запуск, остановку и перезапуск контейнера.

Когда пользователь деплоит проект, `main.py` вызывает `create_student_container(...)`, а этот файл уже напрямую общается с Docker SDK.

**app/templates/index.html**
Главный HTML-шаблон страницы.

В нем описаны сразу три состояния интерфейса:
- неавторизованный пользователь: видит модальное окно входа/регистрации;
- обычный пользователь: видит форму загрузки проекта и список своих проектов;
- администратор: видит админ-панель, пользователей, лимиты, контейнеры и Docker-статус.

Шаблон получает переменные из `main.py`: `user`, `projects`, `users`, `settings`, `limits`, `docker_status`, `error`.

**app/templates/settings.html**
Всплывающая боковая панель настроек.

Сейчас там только:
- email текущего пользователя;
- тип аккаунта;
- кнопка выхода.

Она подключается в `index.html` через:

```jinja2
{% include "settings.html" %}
```

Позже сюда удобно добавлять настройки профиля, смену пароля, тему и т.д.

**requirements.txt**
Список Python-зависимостей проекта.

Там лежат:
- `fastapi` для веб-приложения;
- `uvicorn` для запуска сервера;
- `docker` для управления контейнерами;
- `sqlalchemy` для ORM и работы с БД;
- `psycopg2-binary` для PostgreSQL;
- `jinja2` для HTML-шаблонов;
- `python-multipart` для загрузки файлов и обработки форм.

**docker-compose.yml**
Описывает запуск всей системы.

Сейчас там три сервиса:
- `gateway`: nginx, принимает запросы снаружи;
- `app`: FastAPI-приложение;
- `db`: PostgreSQL.

Также туда передаются переменные:
- `DATABASE_URL`;
- `ADMIN_EMAIL`;
- `ADMIN_PASSWORD`.

**nginx_proxy.conf**
Конфигурация nginx-шлюза.

Обычно он отвечает за маршрутизацию:
- запросы к основному приложению идут в FastAPI;
- запросы к пользовательским сайтам `/s/...` проксируются к нужным контейнерам.

**Dockerfile**
Инструкция сборки контейнера FastAPI-приложения.

Он:
- берет Python-образ;
- ставит системные зависимости;
- устанавливает Python-пакеты из `requirements.txt`;
- копирует код;
- запускает `uvicorn app.main:app`.

Если совсем коротко:  
`main.py` управляет логикой сайта, `db.py` хранит пользователей/проекты/лимиты, `docker_manager.py` запускает контейнеры, `templates` рисуют интерфейс, а Docker Compose поднимает всю инфраструктуру.
